Protocol Handler Test

Test Your Custom Protocol Handler

This page will test your custom open:// protocol handler by attempting to open both a file and a folder.

Note: Make sure that:
  • The protocol handler application has been registered (run as administrator)
  • The file C:\Temp\test.xlsx exists on your system
  • The folder C:\Temp exists on your system

Test 1: Open Excel File

Click the button below to test opening a file:

Open Excel File

Or click this link: open://C:/Temp/test.xlsx

Test 2: Open Folder

Click the button below to test opening a folder:

Open Temp Folder

Or click this link: open://C:/Temp

Troubleshooting

If nothing happens when you click the link:

  • Verify that the protocol handler is properly registered
  • Check that the file exists at the specified location
  • Look at the console output of your protocol handler application for error messages
